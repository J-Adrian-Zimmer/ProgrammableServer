Python Programmable Web Server

The Programmable Server is Python-based and bare bones. It is made to have apps plugged into it.

Here is a list of what is possible without any apps or customization:

  1. Get this screen by entering the abbreviated URL localhost in your browser.
  2. Shut the server down by entering localhost/shutdown in your browser.
  3. See a bird picture by entering localhost/media/Foliage.jpg in your browser.
  4. See a listing of what is in the media directory by entering localhost/media.
  5. Add other things to the public subdirectory so you can see them the way you've seen the bird picture.

Two ways of learning to write apps are:

  1. Study the included demonstration app. Go to the demo_app directory and run install.py with Python 2.7.x. Now when you restart the Programmable Server

    localhost/about

    will show a screen that provides an overview of the demonstration app and explains how to get the original configuration of the server back.

    Many of you can learn to create apps by studying the code that enables these demonstrations.

  2. If step 1 is insufficient, buy the ebooklet “A Python Programmable Web Server”. It will soon be available in Kindle form from Amazon and in epub form from many other sources.

You can customize the behavior of the Programmable Server by editing the config.py file and restarting the server. Comments in config.py explain the many possibilities. Two examples are:

  1. The original configuration only allows access from browsers on the same computer as the server. (Hence the use of localhost rather than a domain name in the above URLs.) You can remove this restriction by setting the localServe parameter to False.

    Note that when you enable requests from browsers on other computers, the shutdown request will still only work with localhost.

  2. You can change the port the server listens on by resetting the port parameter.
